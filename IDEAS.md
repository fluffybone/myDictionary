# IDEAS

## Умное автозаполнение слова

При вводе нового английского слова в разделе `Сейчас учу` можно автоматически подтягивать:

- перевод на русский;
- определение;
- часть речи;
- транскрипцию;
- ссылку на аудио произношения, если есть;
- примеры использования, если есть.

Возможные бесплатные источники:

- `Dictionary API` — определения, части речи, транскрипции, аудио, примеры: https://dictionaryapi.dev/
- `MyMemory API` — простой бесплатный перевод: https://mymemory.translated.net/doc/spec.php

Текущий MVP: автозаполнение запускается после потери фокуса у поля `Новое слово`.

## Вкладка поиска слов

Добавить отдельную вкладку `Поиск`, где пользователь вводит тему или слово, а приложение предлагает слова для изучения.

Примеры сценариев:

- пользователь вводит `travel`, приложение предлагает `journey`, `airport`, `luggage`, `departure`;
- рядом с каждым словом есть кнопка `Добавить`;
- при добавлении можно сразу подтянуть перевод и описание через автозаполнение.

Возможный бесплатный источник:

- `Datamuse API` — поиск похожих слов, ассоциаций и слов по теме: https://www.datamuse.com/api/

## Готовые наборы слов

Сделать локальные подборки слов по темам:

- путешествия;
- работа;
- еда;
- дом;
- эмоции;
- фразовые глаголы;
- базовые слова для начинающих.

На первом этапе их можно хранить как JSON-файлы во фронтенде без внешних API.

## Слово дня

Добавить блок `Слово дня`:

- одно новое слово каждый день;
- перевод;
- короткое определение;
- пример;
- кнопка `Добавить в мой словарь`.

Можно начать с локального списка слов, чтобы фича была стабильной и не зависела от внешних сервисов.

## Умные рекомендации

Предлагать слова на основе уже добавленного словаря:

- похожие слова;
- синонимы;
- слова из той же темы;
- слова, которые часто идут рядом.

Можно реализовать после вкладки `Поиск`, используя те же источники данных.

## Проверка дублей и нормализация

Перед сохранением слова можно улучшить UX:

- приводить слово к нижнему регистру;
- обрезать лишние пробелы;
- предупреждать, если такое слово уже есть;
- предлагать открыть существующую карточку вместо создания дубля.

## Возможный план MVP

1. Оставить текущее автозаполнение слова после `blur`.
2. Добавить backend-ручку `/api/dictionary/autofill?word=hello`, чтобы запросы к внешним API шли через сервер.
3. Добавить вкладку `Поиск`.
4. Подключить поиск слов через `Datamuse API`.
5. Добавить кнопку `Добавить` у найденных слов.
6. При добавлении найденного слова использовать автозаполнение перевода и описания.
